# 节拍器 APK 打包 - 实用方案

## ❌ 为什么 Windows 原生打包失败？

Buildozer 和 python-for-android 在纯 Windows 环境下存在以下问题：
1. Buildozer 在 Windows 上没有 Android target（只有 iOS）
2. python-for-android 依赖的 `sh` 库不支持 Windows

---

## ✅ 实用解决方案（按推荐顺序）

### 方案1：使用 Kivy Launcher（最简单，无需打包！）⭐⭐⭐⭐⭐

**最快方案：直接在手机上运行 Python 代码，无需打包！**

步骤：
1. 在手机上安装 Kivy Launcher
   - Google Play: https://play.google.com/store/apps/details?id=org.kivy.pygame
   - 或 APK下载: https://github.com/kivy/kivy-launcher/releases

2. 将项目文件复制到手机：
   - 复制 `main.py` 和 `sounds/` 文件夹
   - 放到手机 `/sdcard/kivy/节拍器/` 目录

3. 在手机上打开 Kivy Launcher，选择运行！

---

### 方案2：使用 WSL (Windows Subsystem for Linux) ⭐⭐⭐⭐

在 Windows 上运行 Linux，然后在 Linux 中打包：

#### 步骤1：启用 WSL
```powershell
# 以管理员身份运行 PowerShell
wsl --install
```
重启电脑

#### 步骤2：在 WSL 中设置环境
```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装依赖
sudo apt install -y python3-pip openjdk-17-jdk unzip git

# 安装 Python 包
pip3 install --upgrade pip
pip3 install buildozer python-for-android cython kivy numpy

# 设置 Android SDK (如果需要，Buildozer 会自动下载)
# export ANDROID_HOME=$HOME/Android/Sdk
```

#### 步骤3：打包
```bash
# 进入项目目录
cd /mnt/e/new_tdx_mock/PYPlugins/user/JPTool

# 开始打包
echo "y" | buildozer android debug
```

APK 将生成在 `bin/` 目录

---

### 方案3：使用 GitHub Actions 在线构建 ⭐⭐⭐

使用免费的 GitHub Actions 自动构建：

#### 步骤1：创建 GitHub 仓库
1. 将项目推送到 GitHub
2. 我已为您创建了 `.github/workflows/build-apk.yml`

#### 步骤2：触发构建
- 在 GitHub 仓库页面点击 "Actions"
- 选择 "Build Android APK" 工作流
- 点击 "Run workflow"

#### 步骤3：下载 APK
- 构建完成后，在 Actions 页面下载 Artifacts
